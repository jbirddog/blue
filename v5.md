# Blue v5

_RIP: v5 for now_

This is all a Blue frontend issue, BlueVM holds (for now).

## On v4 and a v5

v4 was very close to the goal i think. Close enough that v5 might actually just be v4.1.

One key thing though was the realization that Blue at its core it not so much a full featured Forth that has no
stack after compilation, but a high level assember with Forth semantics.

### Likes

1. Having frontend target BlueVM
1. Blue frontend did not put the BlueVM into compile mode on `:`
1. Dictionary living in the frontend

### Dislikes

1. Frontends have to track BlueVM `here` for addressing

### Needs

1. When compiling `:` should mark an address that can be referred to by number.
   1. Blue frontend needs to build this up
   1. Write these addrs to the first N qwords of the codebuffer
   1. Have opcode to get/set
   1. Frontend makes start return vm start + N qwords
1. Something like `syscall` should be inlined or an immediate word
   1. In this case the initial OF 05 don't really need to land in the code buffer
   1. Frontend needs to handle this
