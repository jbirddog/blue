# v2 Ideas/Changes

## Keep

* stack does not exist after compile time
* developer should have full access to registers/memory as if writing asm
* still x8664/linux centric but keep in mind ways to change that
* still relying on an external assembler for now

## Change/Improve

* reduce the need to declare registers
* opcodes need to be defined as "normal" words instead of being built into the compiler
* no opcodes/syscalls are defined by default
* words have access to the compilation buffer, first pass will continue to write asm
* first pass, remove the optimizer since some language features rely on it
* remove the concept of indirect registers
* add ability to push/pop from the cpu stack (return stack)
* some form of multiple dispatch needs to be supported per word, consider a subset of `add`:

| Opcode | Instruction |
|----|----|
| 04 ib | ADD AL, imm8 |
| 05 id | ADD EAX, imm32 |
| 00 /r | r/m8, r8 |
| REX.W + 01 /r | ADD r/m64, r64 |
